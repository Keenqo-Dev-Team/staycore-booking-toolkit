## Pourquoi un site de réservation directe Stay’Core ?

Quand un voyageur réserve via **Airbnb** ou **Booking.com**, ces plateformes te prélèvent **15 à 20 % de commission**. Avec un site de réservation directe connecté à Stay’Core, le voyageur paye le même prix → mais **tu gardes 100 % du montant**.

- 🎯 **Zéro commission** OTA (Airbnb, Booking.com, Expedia)
- 💳 **Paiement Stripe** intégré (cartes, Apple Pay, Google Pay)
- 📅 **Calendrier temps réel** synchronisé avec ton calendrier Stay’Core et tous tes channels
- 🛡️ **Caution Stripe Connect** pré-autorisée
- 📧 **Email de confirmation** automatique + suivi de réservation par token
- 🧪 **Mode test** pour valider ton flow avant la mise en prod
- 🎨 **4 presets** visuels (love-room, château luxe, gîte nature, city business) ou direction artistique 100 % custom

## Ce que tu obtiens en `npx create-staycore-site`

```bash
npx create-staycore-site@latest mon-site \
  --org-slug=mon-org \
  --preset=love-room
```

- Un projet **Vite + React + TypeScript + Tailwind** prêt à déployer sur Vercel / Netlify / Cloudflare Pages
- Le **moteur de réservation Stay’Core** déjà branché (calendrier, prix dynamique, Stripe checkout, paiement caution)
- Tes **vraies propriétés** pré-remplies depuis le PMS au moment du scaffold (live API)
- **GA4 Consent Mode v2** RGPD-compliant
- **JSON-LD `LodgingBusiness`** + meta Open Graph sur chaque fiche
- **Banner mode test** automatique tant que `test_mode_enabled` est ON

## Architecture

```
@staycore/booking-sdk        ← Client TypeScript + hooks React (npm)
create-staycore-site         ← CLI de scaffolding (npx)
staycore-direct-booking      ← Skill Claude Code (marketplace + git)
/api/v1/book/{org_slug}/*    ← API publique versionnée, contrat figé
```

Tous les chemins consomment **la même API publique versionnée** `https://api.stay-core.com/api/v1/book/{slug}/*`. Le contrat est gelé : aucune évolution breaking sans bump majeur du SDK, garanti par un test PHPUnit qui fail le CI au moindre changement.
